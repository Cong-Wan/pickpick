### 问题清单

#### 🟡 [Medium] `rawConverter.cpp` 中 `bright=1.0f` 在 auto-bright 模式下语义冗余

**位置**: `cpp/src/rawConverter.cpp:57`
**问题**: `no_auto_bright = 0` 后，`bright` 参数在 LibRaw 的 auto-bright 模式下**被忽略**（auto-bright 会覆盖 bright 的线性乘法逻辑）。保留 `bright = 1.0f` 虽然无害，但会让后续维护者误以为它在生效，造成困惑。
**修复方案**:
```cpp
// 建议添加注释说明，或移除 bright 赋值
rawProcessor.imgdata.params.no_auto_bright = 0;
// bright 在 auto-bright 模式下被忽略，保留默认值
// rawProcessor.imgdata.params.bright = 1.0f;
```

#### 🔵 [Low] `pinchStartMagnification` 在手势结束后未重置

**位置**: `rawViewer/views/metalPhotoView.swift:86`
**问题**: 变量在 `.ended` 后仍保留旧值，直到下一次 `.began` 才被覆盖。这不是 bug，但语义上不够干净。如果未来代码在 `.ended` 后误读取此变量，会产生误导。
**修复方案**:
```swift
default:
    pinchStartMagnification = 0.0
    break
```
