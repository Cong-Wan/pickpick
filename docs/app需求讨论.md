# 需求讨论

当前的项目是我使用 xcode 创建的，我想开发一个 macOS 的 app，UI 部分我想使用 app kit 框架；

我使用 cpp 写了一些算法，代码在 @cpp 文件夹中，它所实现的功能在 @docs/recipe/photo-analyzer-design.md 文档中，先去阅读；

在开始描述我的软件设计之前，我需要你回答我的问题：

软件层面怎么调用 cpp 的程序？是通过函数的调用吗？还是通过模拟命令行执行 cpp 的二进制文件呢？我希望你提供一个优雅且简洁的方案！

软件的设计说明如下；

1. 程序打开的首页应该是一个虚线框，虚线框内有个灰色的加号，然后还有一行小字，大致意思就是提示用户可以点击加号选择文件夹，也可以将文件夹拖入其中；这个虚线框的大小应该有个最大值和最小值，随着窗口变化而变化；
2. 当用户上传了一个文件夹后，应该去调用 cpp 程序进行分析（怎么调用待你回答我上面的问题），然后转圈，显示进度，提示用户正在分析；
3. 当分析完成之后应该有个分组（因为 cpp 分析会输出 “过/欠曝光、虚焦、正常、重复组” 的分类，注意：这里重复组的代码 cpp 还没有实现，计划是将重复的照片归为一组，所以这里可能有多个重复组）；
   1. 用户随便点进去一个组，都得显示这个组里的照片，这里我又有疑问了，我期望的是使用 cpp 开发这个读取照片的部分，然后软件侧通过零拷贝的方式拿到读取的数据显示处理，这样能快速的读取照片，用起来更丝滑，就是这部分该怎么实现？
   2. 先说共性的部分，不管哪一个组用户点进去，中间部分都以大图的形式显示第一张照片，上下键可以切换，blackspace 键是删除（需要弹出二次确认框）；然后左侧以列的形式展示所有图片（原图，只是尺寸不需要那么大），然后左上角有个选择框，用户可以点击多个图片的选择框（左侧预览的上方有个全选的小选择框）然后点击删除；
